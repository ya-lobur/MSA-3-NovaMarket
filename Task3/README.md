# Задание 3. Масштабируйте приложение под нагрузку

Сейчас сервисы **NovaMarket** развёрнуты в Kubernetes. Каждый сервис запущен в определённом количестве экземпляров.
Обычно этого достаточно для стабильной обработки запросов.

Однако в периоды пиковой нагрузки система не справляется и демонстрирует нестабильность — поды постоянно перезагружаются
из-за нехватки памяти. Это приводит к негативному пользовательскому опыту, и бизнес фиксирует снижение **NPS**.

Можно было бы постоянно держать увеличенное количество реплик для обработки пиковых нагрузок. Однако это экономически
невыгодно и приводит к низкой утилизации ресурсов.

Поэтому необходимо решить проблему с помощью **динамического масштабирования** сервисов.

---

## Тестовое приложение

Для тестирования используется простое приложение, которое предоставляет два ресурса:

- `GET /` — получение идентификатора пода
- `GET /metrics` — получение метрик в формате Prometheus

Метрика `http_requests_total` возвращает количество запросов к методу получения идентификатора пода.

**Образ тестового приложения:** `ghcr.io/yandex-practicum/scaletestapp:latest`

Оба метода приложения доступны по порту `8080`.

# Динамическое масштабирование на основании утилизации памяти

## Поднимите локальный кластер Kubernetes в Minikube.

1. Активируйте metrics-server.
2. Напишите манифест развёртывания (Deployment) Kubernetes для запуска тестового приложения. Для начального количества
   реплик установите значение, равное единице. Лимит памяти — равным 30Mi. Примените написанную конфигурацию в вашем
   кластере. Файл с манифестом добавьте в директорию Task3 вашего репозитория.
3. Напишите и примените манифест сервиса (Service) для доступа к приложению, которое вы установили на прошлом шаге. Файл
   с этим манифестом тоже загрузите в директорию Task3.
4. Настройте динамическую маршрутизацию на основании показателей утилизации оперативной памяти с помощью Horizontal Pod
   Autoscaler (HPA). Для вашего тестового приложения оптимальный уровень утилизации памяти равен 80%. В качестве
   максимального количества реплик рекомендуем установить 10. Примените манифест в вашем кластере. Загрузите готовый
   манифест в директорию Task3.
