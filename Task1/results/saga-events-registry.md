# Реестр событий Saga-хореографии оформления заказа

## Описание Saga

Saga-хореография для оформления заказа включает следующие основные этапы:

1. Создание заказа
2. Резервирование товаров на складе
3. Обработка платежа
4. Создание заявки на доставку
5. Отправка уведомлений

При возникновении ошибок на любом из этапов запускаются компенсационные действия для отката предыдущих успешных
операций.

## Таблица событий

| Этап                                     | Тип события  | Название события           |
|------------------------------------------|--------------|----------------------------|
| Создание заказа                          | domain       | OrderCreated               |
| Успешное резервирование товаров          | domain       | InventoryReserved          |
| Ошибка резервирования товаров            | failure      | InventoryReservationFailed |
| Отмена заказа при неудаче резервирования | compensation | OrderCancelled             |
| Успешная оплата                          | domain       | PaymentSucceeded           |
| Ошибка оплаты                            | failure      | PaymentFailed              |
| Освобождение зарезервированных товаров   | compensation | InventoryReleased          |
| Отмена заказа при неудаче оплаты         | compensation | OrderCancelled             |
| Успешное создание заявки на доставку     | domain       | DeliveryRequestCreated     |
| Ошибка создания заявки на доставку       | failure      | DeliveryRequestFailed      |
| Возврат средств                          | compensation | PaymentRefunded            |
| Освобождение зарезервированных товаров   | compensation | InventoryReleased          |
| Отмена заказа при неудаче доставки       | compensation | OrderCancelled             |
| Успешная отправка уведомления продавцу   | domain       | SellerNotificationSent     |
| Успешная отправка уведомления покупателю | domain       | CustomerNotificationSent   |
| Обновление статуса заказа                | domain       | OrderStatusUpdated         |
| Завершение заказа                        | domain       | OrderCompleted             |

## Дополнительные события для мониторинга

| Этап                         | Тип события | Название события           |
|------------------------------|-------------|----------------------------|
| Изменение цены товара        | domain      | ProductPriceChanged        |
| Изменение доступности товара | domain      | ProductAvailabilityChanged |
| Изменение статуса доставки   | domain      | DeliveryStatusChanged      |
| Подтверждение привязки карты | domain      | CardBindingConfirmed       |

## Примечания

- **domain** - события, отражающие успешное завершение бизнес-операции
- **failure** - события, сигнализирующие об ошибке в процессе выполнения операции
- **compensation** - события, запускающие откат ранее выполненных операций

Все события содержат идентификатор заказа (orderId) для обеспечения корреляции между различными этапами Saga.
